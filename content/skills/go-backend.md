Go es mi lenguaje principal para backend, con más de 7 años de experiencia diseñando APIs REST
(verbos HTTP, middleware, rate limiting, autenticación con tokens JWT) y sistemas concurrentes con
patrones de worker pools, fan-in/fan-out, productor-consumidor, pipelines y future/promise,
manejando el contexto para controlar cancelaciones y timeouts. También me ocupo de testing,
optimización de rendimiento y observabilidad (métricas con Prometheus y Grafana, OpenTelemetry),
incluyendo integraciones con APIs externas como Microsoft Graph y el diseño de sistemas
resilientes ante su rate limiting. Antes de asentarme en Go trabajé con Java (POO, Garbage
Collector, patrones de diseño, y algo de Spring), y también conozco Python y Node.js con su
modelo de concurrencia asíncrono basado en event loop.
